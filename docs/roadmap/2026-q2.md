# Q2: Platform (Apr -- Jun 2026)

The second quarter transforms ClawBio from a repository into a platform with a web hub, authoring tools, automated validation, and seamless agent-framework integration.

## Key Deliverables

### clawbio.ai Hub

- **Registry**: searchable catalogue of all verified skills
- **Skill cards**: metadata, domain, validation tier, install command, demo output
- **One-click install**: copy-paste CLI command or direct Claude Code plugin install
- **Search**: full-text + faceted filtering by domain, guideline authority, validation tier

### Authoring Toolkit

- `clawbio init` -- scaffolds a new skill directory with SKILL.md template, demo data stub, and test harness
- `clawbio validate` -- local validation against the JSON schema, runs tests, checks demo data
- `clawbio publish` -- submits skill to the registry for review and publication
- **VS Code extension** with syntax highlighting for SKILL.md frontmatter and inline validation

### Automated Validation Pipeline

- GitHub Actions workflow triggered on every skill PR
- Docker-based execution: skills run in isolated containers
- Validation checks: schema compliance, demo data runs, output format, safety rules
- Badge system: passing / failing / pending review

### MCP Bridge

- Every skill auto-exposes as an MCP (Model Context Protocol) server
- Agent frameworks can discover and call skills via MCP without custom integration
- No SDK required -- standard MCP protocol

### DOI Minting

- Live DOI minting via Zenodo integration
- Every published skill receives a persistent DOI on first release
- Version DOIs for subsequent updates
- Citation metadata automatically generated

### Registry API

- RESTful API documented with OpenAPI spec
- `GET /api/v1/skills` -- list and search skills
- `POST /api/v1/run/{skill_name}` -- invoke a skill remotely
- Authentication via API key

### Galaxy Interoperability

- RO-Crate export for every skill (Research Object packaging)
- EDAM ontology alignment for domain and operation terms
- Galaxy Tool Shed compatibility layer
